# Knowledge index

This folder holds **distilled** notes, not raw material — and it's not limited to
academic papers, and not limited to vLLM parameter tuning. Anything that should inform
how studies are designed or run (papers, articles, blog posts, vendor docs, conference
talks) gets distilled the same way, across four kinds of source: **(1)** vLLM/GPU/
Kubernetes parameter tuning, **(2)** serving tooling & platforms (e.g. llm-d, NVIDIA
Dynamo, Triton Inference Server, KServe), **(3)** load-testing tools & methodology for
inference (e.g. GuideLLM, GenAI-Perf/AIPerf, trace-replay vs. synthetic load), and **(4)**
Kubernetes/infra patterns for GPU workloads (device plugins, MIG/time-slicing/MPS, DRA,
GPU-aware autoscaling, multi-tenancy). A source doesn't need to map onto a tunable Akamas
parameter to earn a note — a tooling or infra pattern that changes how a future study
should be scoped is just as valuable. Raw sources go in `knowledge/sources/` (gitignored
content is fine — PDFs, HTML dumps; for an ordinary web article, a link in the note is
usually enough, no need to mirror the whole page) and get *distilled* into a short `.md`
note under `knowledge/notes/` using the template below. Nothing gets loaded into an
Akamas study or the ROADMAP straight from a raw source — it passes through a note first.

Use **`/update-knowledge`** (`.claude/commands/update-knowledge.md`) to add a new note —
give it a URL, a pasted excerpt, or a local file, and it handles fetching, distilling
into `notes/_TEMPLATE.md`'s shape, and updating the index below.

Every note uses the same five fields (see `notes/_TEMPLATE.md`):
problem addressed, levers/parameters touched, key results, implications for vLLM/k8s
tuning, which Akamas parameters to explore.
